# Contributing

Thank you for helping improve SEEH.

## Goals

-   Keep documentation beginner friendly.
-   Test every procedure before submitting.
-   Explain *why* as well as *how*.
-   Follow established naming conventions.

## Pull Requests

Please include:

-   Description of changes
-   Testing performed
-   Screenshots if applicable

## Style

-   Use Markdown.
-   One task per step.
-   Include verification and troubleshooting sections.
